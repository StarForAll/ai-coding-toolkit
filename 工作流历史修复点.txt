1. 前置校验:
使用该工作流的项目必须是一个git项目且存在多个remote仓库,其对应添加远程仓库参考命令如下:
git remote set-url --add --push origin git@github.com:xxx/yyy.git
git remote set-url --add --push origin git@gitee.com:xxx/yyy.git

2. 初始阶段导入的spec应该是使用对应的脚本进行导入而非用户输入提示进行安装

3. trellis init创建的任务00-bootstrap-guidelines在安装这个工作流之后需要删除，finish-work中的 pnpm校验删除，后续技术架构确定之后补充明确的内容

4. 在Brainstorm阶段确认完成之后需要在当前项目下的docs目录中生成完全可用的两份项目级需求文档文档；一份面向客户，一份面向开发人员

5. 开启新的任务时需要判断对应的任务是否要求前一个任务必须完成，如果是则需要先进行校验，如果前一个任务完成了才真正开始当前任务

6. 初始阶段校验当前分支是main分支且在Brainstorm阶段结束之后添加的门禁中补充使用 sonar-scanner 

7. 任务信息分配给具体的任务不能挤压到 task_plan.md 文件中而是使用trellis的任务机制且每一个具体的任务需要在执行前补充其任务相关的测试门禁

8. 项目包含多种文档以及禁止codex使用原型实现前端界面,design阶段需要尽可能完善在stitch生成原型所需要的提示词
