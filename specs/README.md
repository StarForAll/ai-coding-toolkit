# Specs

这里存放**具体编码规范文件**（规范本体），用于沉淀“可复用、可审阅、可版本化”的约定基线。

## 建议组织方式

```text
specs/
  README.md
  <topic>/
    README.md          # 本主题规范的适用范围与要点
    <rule>.md          # 具体规范条目（可拆分为多个文件）
```

常见主题示例（按需创建）：

- `specs/git/`：分支/提交信息/PR 规范
- `specs/frontend/`：前端工程与组件约定
- `specs/backend/`：后端分层、接口、错误码等约定
- `specs/testing/`：测试策略与覆盖要求
- `specs/security/`：安全基线与敏感信息处理

## 编写约定（建议）

- 以“可执行”为导向：写清楚适用范围、必须/禁止项、例外情况、验证方式。
- 尽量给出可验证的命令/检查项（lint/test/CI gate），避免纯口头约定。
