# Normative Rules

* A handoff must preserve enough context for the receiver to continue without reconstructing the full task from scratch.
* Handoffs should separate confirmed facts, open questions, risks, and recommended next actions.
* File ownership, write scope, and unresolved conflicts must be explicit when code changes are involved.
* A handoff must not describe work as complete when verification is still pending or absent.
* If the receiver cannot safely continue from the handoff, the handoff is incomplete.
