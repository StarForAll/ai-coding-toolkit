# Normative Rules

* Ambiguous requirements must be clarified before they are treated as settled implementation commitments.
* Blocking unknowns must be separated from non-blocking preferences.
* Clarification should prefer evidence from repo context, docs, and research before asking stakeholders for avoidable input.
* Conflicting statements must be surfaced explicitly instead of silently reconciled by guesswork.
* If requirements remain unstable, the workflow must reflect that instability in scope and confidence.
