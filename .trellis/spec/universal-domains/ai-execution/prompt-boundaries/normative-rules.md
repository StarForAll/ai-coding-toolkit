# Normative Rules

* Stable rules, terminology, and non-changing context must be separated from dynamic task details.
* Prompt content must not include irrelevant historical context when a smaller stable summary is available.
* Prompt composition must preserve clear boundaries between instruction, evidence, assumptions, and requested output.
* Project-private or sensitive data must not be injected into reusable prompt assets unless explicitly required and allowed by policy.
* Prompt updates must prefer minimal changes over uncontrolled prompt growth.
