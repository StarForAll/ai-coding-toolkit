# 近期改进：需求变更管理流程避免形式化

## 目标

调整工作流总纲 §2.5 需求变更管理的结构顺序，让"估价-报价-确认"成为第一优先的快速闭环，避免流程形式化。

## 现状问题

1. **顺序不合理**：当前是"评估→定价→审批→执行"，但客户第一反应是问"多少钱"
2. **文档结构混乱**：8步流程是默认，但轻量变更单（2.5.6.1）藏在最后
3. **估价和定价分离**：2.5.5 定价原则没有整合 2.5.5.1 的阶段系数表

## 改动方案

### 改动一：调整 §2.5.3 流程顺序

**当前顺序**（8步）：
```
① 变更接收 → ② 变更评估 → ③ 变更方案制定 → ④ 变更评审 → ⑤ 变更审批 → ⑥ 变更执行 → ⑦ 变更验证 → ⑧ 变更关闭
```

**调整后**：把"估价-报价-确认"作为前置快速闭环，后续保留完整流程作为可选延伸

```
① 快速估价（新增：30分钟内口头/书面估价）
    └── 用阶段系数表快速估算

② 客户确认意向（新增）
    └── 接受则进入后续流程，拒绝则记录关闭

③ 变更接收与记录（现行 ①）
④ 变更评估与方案（现行 ②③ 合并）
⑤ 变更评审（现行 ④）
⑥ 变更审批（现行 ⑤）
⑦ 变更执行（现行 ⑥）
⑧ 变更验证（现行 ⑦）
⑨ 变更关闭（现行 ⑧）
```

### 改动二：调整 §2.5.5 定价原则位置和内容

**调整**：把 2.5.5 定价原则整合 2.5.5.1 阶段系数表，形成统一的估价参考

**新增**：在 2.5.5 中增加一个"快速估价公式"：

```
变更估价 = 基准工时 × 阶段系数 × 复杂度系数 + AI资产附加

其中：
- 阶段系数：需求冻结前 1.0x → 交付后 3.0x+
- 复杂度系数：简单 1.0x / 一般 1.2x / 复杂 1.5x
- AI资产附加：按 Prompt/评估集/上下文影响估算
```

### 改动三：前置轻量变更单模板

**调整**：把 2.5.6.1 最小变更单模板**前置到 2.5.3 之后**，作为"快速闭环推荐模板"
