# Normative Rules

* Refactoring should preserve declared behavior unless behavior change is made explicit.
* Large refactors should be decomposed into verifiable steps when possible.
* Behavior-preservation claims must be supported by tests, checks, or explicit evidence.
* Refactoring should not silently absorb unrelated feature work.
* If the refactor introduces new patterns worth reusing, the relevant reusable specs should be updated.
