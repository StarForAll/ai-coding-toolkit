# Normative Rules

* Review feedback must be understood and validated before implementation, not blindly applied.
* Accepted review changes must be checked for secondary effects and new regressions.
* Review-driven changes should preserve the original task context instead of becoming disconnected patchwork.
* If feedback reveals a reusable lesson, the relevant reusable spec should be updated.
* Disputed or unclear feedback should be resolved explicitly rather than silently reinterpreted.
