# Normative Rules

* Work should be handed off only when current state, blockers, and next actions are explicit enough for safe continuation.
* Claimed progress must distinguish completed work, in-progress work, and unverified assumptions.
* File ownership, pending verification, and known risks must be preserved before ownership changes.
* A handoff should not force the receiver to rediscover basic task boundaries that are already known.
* If context is too fragmented to transfer safely, readiness should be treated as not yet achieved.
