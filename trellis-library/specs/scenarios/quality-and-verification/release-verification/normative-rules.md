# Normative Rules

* Release verification must reflect the actual release scope, affected risk areas, and exposure plan.
* Evidence should be assembled before release claims are made, not reconstructed after issues appear.
* Known verification gaps, manual-only checks, and deferred risks must remain visible at release decision time.
* Release verification should confirm not only correctness but also observability, rollback readiness, and dependency assumptions where relevant.
* A release candidate with unclear evidence boundaries must not be described as fully verified.
