# Normative Rules

* Interactive browser experiences should remain usable without relying solely on pointer precision, visual cues, or uninterrupted motion.
* Semantic structure and state changes must be exposed clearly enough for assistive technologies to interpret meaningful behavior.
* Keyboard, focus, and announcement behavior should be treated as first-class interaction paths where web interaction depends on them.
* Accessibility support must remain part of normal behavior design rather than an afterthought once visual implementation is already fixed.
* When accessibility coverage is incomplete, the remaining user impact should remain visible rather than implied away.
