# Normative Rules

* Meaningful changes must be assessed before implementation continues under the new assumption.
* Changes must be classified by impact on scope, risk, dependencies, verification burden, and delivery commitments.
* Approved changes must update the active task context, acceptance criteria, and verification expectations.
* Unapproved changes must not be smuggled into delivery as incidental implementation details.
* Rejected or deferred changes must remain visible rather than silently disappearing from the record.
