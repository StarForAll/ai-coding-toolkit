# Normative Rules

* Changes that address known failures or high-risk behavior must define how recurrence will be detected or prevented.
* Regression protection should target the failure mode or behavior, not only the specific code path touched.
* Repeated failure patterns must raise the expected strength of regression safeguards.
* If regression protection cannot be automated, the remaining exposure must be made explicit.
* Regression expectations must remain aligned with current acceptance and verification criteria.
