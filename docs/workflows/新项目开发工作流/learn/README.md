# learn 目录说明

> 用于沉淀 `新项目开发工作流` 在真实使用过程中暴露的流程问题、经验和优化候选项。
>
> **注意**：本目录只存放**已经确认需要保留**的经验反馈记录。AI 起草的草稿应先放在项目的 `tmp/` 目录下，由用户确认后移入本目录。完整起草流程见 `工作流总纲.md §7.3.1`。

---

## 目的

这个目录解决的不是"项目本身怎么复盘"，而是"当前工作流作为一套方法论，在实际使用时哪里有摩擦、歧义、漏项和可优化空间"。

典型记录来源：

- 流程执行中遇到的卡点、返工、误判
- 命令说明不清、触发词歧义、阶段路由错误
- `delivery`、`review-gate`、`record-session` 等阶段暴露出的闭环问题
- 人工介入后确认属于流程/规范层的问题
- 复盘中发现的可复用经验、提示词模式、反模式

---

## 使用原则

### 1. 文件是一次性的

learn 中的每条记录都是**一次性使用**的：

1. 人工复盘时读取记录
2. 决定是否修改工作流文档
3. 改了就在文件末尾补一行回流记录
4. 文件使命完成，不需要长期维护状态或索引

### 2. AI 可以归纳，人工决定是否回流

AI 可以：

- 起草经验记录（放在项目 `tmp/` 目录）
- 归纳重复模式
- 提出工作流优化建议

但以下动作必须人工确认后再执行：

- 修改 `工作流总纲.md`
- 修改命令路由、阶段门禁或默认推荐路径
- 把临时经验升级为长期规范

### 3. 优先沉淀可复用模式

优先记录那些能够跨任务、跨项目、跨阶段复用的经验，而不是一次性噪音。

## 文件命名约定

```text
YYYY-MM-DD-主题短名.md
```

例如：

```text
2026-03-31-learn-feedback-mechanism-sample.md
2026-04-02-phase-router-misroute.md
```

---

## 写法建议：像开发复盘，不像制度报告

建议按真实开发顺序写：

1. 当时发生了什么
2. 你为什么会卡住 / 误判 / 漏掉
3. 最后是怎么发现和修正的
4. 这个问题是你个人失误，还是工作流本身也容易诱发

能一句话说清的地方不要硬写成长篇。

比起"流程优化建议如下"，更有用的写法通常是：

- "我以为 `record-session` 到这一步就能直接跑，结果 `.trellis/tasks` 还没 clean，来回查了两次才发现得先 archive。"
- "`delivery` 里虽然提了复盘，但没有明确让我把流程层问题单独记到 `learn/`，所以第一次做的时候我只写进了 `retrospective.md`。"

## 值得学习优化的判断标准

至少满足以下 2 项，才建议进入"工作流优化候选"：

1. **重复性**：同类问题出现至少 2 次，或高度可能再次出现
2. **影响面**：会导致误判、返工、漏检、错误路由、交付风险，或明显拖慢流程
3. **可复用性**：修正后能帮助后续多个任务/项目，而不是只对单一场景有效
4. **可执行性**：能够落实为明确的文档、检查项、模板、命令说明或阶段规则调整
5. **收益明显**：优化成本可控，且收益高于维护成本

通常**不建议**进入工作流优化的情况：

- 单次偶发环境问题，且无复现规律
- 纯个人偏好，不具备团队或跨项目复用价值
- 已有记录且本次没有新增证据或新增结论
- 建议本身仍停留在模糊口号，无法转成可执行改动

## 推荐流程

```text
1. AI 在 tmp/ 中起草反馈文档（见工作流总纲 §7.3.1）
   ↓
2. 用户确认后将文件移动到本目录 (learn/)
   ↓
3. 项目复盘或阶段性回顾时，人工读取 learn/ 中的记录
   ↓
4. 对每条记录判断：改工作流 / 不改
   ↓
5. 若确认要改，人工批准后修改工作流文档/命令
   ↓
6. 在原记录末尾补一行回流结果
   ↓
7. 文件使命完成
```

---

## 文档模板

请基于同目录下的 [TEMPLATE.md](./TEMPLATE.md) 新建记录。

模板提供两种格式：
- **快速捕获**：30 秒速记（踩坑位置 + 现象 + 可能原因），适合开发中快速记录
- **完整记录**：起草阶段（现象 + 详细描述 + 建议改哪里）+ 结论阶段（人工结论 + 回流记录）

---

## 维护要求

- 不要把 `learn/` 当作临时草稿堆；无结论的记录至少要保留事实和证据
- 相同问题优先补充既有记录，避免重复文件泛滥
- 若某条记录已完成回流优化，请在原文档末尾补一行回流记录即可，不需要维护复杂的索引或状态
