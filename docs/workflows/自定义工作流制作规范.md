# 自定义工作流嵌入 Trellis — 制作规范

> 从 ai-coding-toolkit 项目实践中提取的约束条件。
> 用于指导「为 Trellis 制作自定义工作流」的完整流程。
> 最后更新: 2026-03-21

---

## 项目定位

```
当前项目是「工作流制作工具」，不是嵌入工作流的目标项目。
install/uninstall/upgrade 脚本是模板，供其他项目使用。
源文件统一放在 docs/workflows/<工作流名>/commands/ 下。
当前项目的 .claude/ 和 .trellis/ 不应被修改。
```

---

## 条件总览

| # | 条件 | 一句话 |
|---|------|-------|
| 1 | 基于 trellis-meta | 遵循 YAML frontmatter + Markdown 格式 |
| 2 | 源文件在工作流子目录 | 不直接放 .claude/，通过脚本部署 |
| 3 | 复用现有 Skills | 每个命令声明 skill 引用关系 |

| 8 | 自动发现项目根 | 不硬编码路径 |
| 9 | 跨 CLI 兼容 | Claude / Cursor / OpenCode / Codex |

| 11 | 不依赖 trellis-local | 安装记录存 .trellis/workflow-installed.json |

## 条件 1：基于 trellis-meta 规范

所有新增命令/skill 必须遵循 trellis-meta 的定义格式。

```
命令文件：YAML frontmatter（name, description）+ Markdown 内容
参考：~/.agents/skills/trellis-meta/references/how-to-modify/
```

## 条件 2：源文件放在工作流子目录

```
项目根/
└── docs/workflows/<工作流名>/
    ├── <工作流文档>.md
    ├── 命令映射.md
    └── commands/
        ├── command-a.md          # 命令源文件
        ├── command-b.md
        ├── start-patch-*.md      # start.md 注入块（独立文件）
        ├── install-workflow.py   # 一键安装
        ├── uninstall-workflow.py # 一键卸载
        ├── upgrade-compat.py     # 版本升级兼容
        ├── shell/                # 辅助脚本
        │   └── helper.py
        ├── cursor/README.md      # Cursor 适配
        ├── opencode/README.md    # OpenCode 适配
        └── codex-gemini/README.md
```

目标项目安装后生成：

```
.claude/commands/trellis/         # 命令
.trellis/scripts/workflow/       # 辅助脚本
.trellis/workflow-installed.json # 安装记录
```

## 条件 4：自然语言触发词

description 字段和文档中必须包含用户实际会说的话。

```yaml
---
name: my-command
description: 功能描述。触发词：用户会说的词1、词2、词3
---
```

文档中展开触发词表：

```markdown
| 用户说 | 触发命令 |
|-------|---------|
| "自然语言1" "自然语言2" | `/trellis:xxx` |
```

同时在 start.md 中维护全局触发词路由表。

---

## 条件 5：分支式下一步推荐

每个命令执行完毕后，AI 必须在末尾输出「下一步推荐」区块（不只线性路径）。

```markdown
## 下一步推荐

**当前状态**: <一句话描述当前完成的阶段和产出>

根据你的意图：

| 你的意图 | 推荐命令 | 说明 |
|---------|---------|------|
| 默认路径 | `/trellis:xxx` | **默认推荐**。<说明> |
| 回退修改 | `/trellis:yyy` | 回到上一步 |
| 跳过前进 | `/trellis:zzz` | 跳过中间步骤 |
| 其他操作 | `python3 script.py` | 或 Skill |
| 不确定下一步 | `/trellis:start` | 用 Phase Router 自动检测 |
```

规则：
- 始终包含「不确定 → `/trellis:start`」兜底
- 给出多种可能路径（回退、跳过、特殊情况）
- 标记**默认推荐**
- 可以推荐自定义命令、Trellis 官方命令、或 Skills

## 条件 6：版本升级兼容

必须提供三件套：`install` → `uninstall` → `upgrade-compat`

### upgrade-compat.py

```
--check  检测冲突（版本比较 + 文件缺失检测）
--merge  自动合并（重新部署命令 + 注入 Phase Router）
--force  强制覆盖（从备份恢复）
```

### 关键设计

- **唯一冲突点是 start.md**（注入了 Phase Router）
- 其他命令都是独立文件，不与官方冲突
- Phase Router 注入块独立为 `start-patch-phase-router.md`
- 安装记录存 `.trellis/workflow-installed.json`（JSON 格式）

### 版本追踪

```json
// .trellis/workflow-installed.json
{
  "trellis_version": "0.4.0-beta.5",
  "installed": "2026-03-21T...",
  "commands": ["feasibility", "design", ...],
  "scripts": ["helper1.py", ...]
}
```

## 条件 7：Python 实现 + 零外部依赖

```
- #!/usr/bin/env python3
- 仅标准库：pathlib, shutil, subprocess, argparse, re, os, sys, datetime, json
- 不 pip install，不修改系统 Python
- 兼容 /usr/bin/python3（系统默认）
- chmod +x
- 颜色输出用 ANSI 转义码（硬编码常量）
- 绝不使用：requests, colorama, rich, click 等第三方库
```

## 条件 8：项目根目录自动发现

脚本不能硬编码路径。

```python
def find_root() -> Path:
    cur = Path(__file__).resolve().parent
    for _ in range(10):
        if (cur / ".claude").is_dir():
            return cur
        if cur.parent == cur:
            break
        cur = cur.parent
    sys.exit("未找到 .claude/ 目录，请在 Trellis 项目内运行")
```

## 制作检查清单

```
□ 源文件在 docs/workflows/<name>/commands/
□ 命令有 YAML frontmatter（name, description 含触发词）
□ 引用了相关 existing skills

□ 有 install-workflow.py（一键安装）
□ 有 uninstall-workflow.py（一键卸载）
□ 有 upgrade-compat.py（版本升级兼容）
□ 有 start-patch-phase-router.md（注入块独立文件）

□ 卸载后目标项目恢复原始状态
□ 跨平台适配器 README 已创建
```
